# mewmew

## Design principles

- **Mobile-first, touch-first.** Every decision is optimized for mobile use with a touch-first design. Desktop is secondary.
- **Never verbose.** Keep the UI minimal. If an instruction can be replaced with an icon, use an icon.
- **Assume expert users.** Treat the interface as if users already know how everything works — no hand-holding, no explanatory text.

## Working with Claude

- **Always ask to push to `main` on GitHub** after making changes.
- **Flag the impossible.** Always notify the user if an implementation seems impossible or infeasible before attempting it.
- **Concise CLI output.** Keep responses in the Claude CLI short. Get straight to the implementation.
